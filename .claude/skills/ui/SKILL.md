# UI Design Skill

Use this skill for general visual reasoning. In DevZeros, defer to `devzeros-design-system` for actual product tokens and component rules.

## Core Principles

- hierarchy before decoration
- product fit before trend chasing
- consistency before novelty
- contrast and readability before atmosphere
- one dominant composition idea per screen

## Color

- prefer existing project tokens over invented palettes
- use accent color sparingly to direct attention
- do not create rainbow surfaces or random premium gradients
- choose color based on function: action, status, structure, or emphasis

## Typography

- use typography to separate hierarchy, not just to add personality
- in DevZeros, UI typography is based on `Space Grotesk` and code typography on `JetBrains Mono`
- avoid generic corporate fallback aesthetics unless the product already uses them

## Layout

- optimize for scan paths and task flow
- let spacing create calm structure
- use asymmetry only when it improves the composition or narrative
- avoid filler sections, filler cards, and filler side panels

## Responsive Design

- mobile should preserve structure, not collapse into a meaningless stack
- desktop should use width to improve comprehension, not to spread content thin
- keep touch targets and focus visibility intact across breakpoints

## Product UI Heuristics

- operational surfaces need clarity first
- trust-sensitive surfaces need calm and confidence
- showcase surfaces can be more expressive, but still need discipline
- if a UI element has no job, remove it
